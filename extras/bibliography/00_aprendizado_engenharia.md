# Aprendizado, metodologia e engenharia

## Metacognição e hábitos

- Barbara Oakley — *Mind for Numbers* / curso **Aprendendo a aprender** (Coursera, PT).
- Make It Stick (Brown, Roediger, McDaniel) — evidência sobre prática recuperativa e espaçamento.

## Git e fluxo de trabalho

- Scott Chacon, Ben Straub — [Pro Git](https://git-scm.com/book/pt-br/v2) (gratuito on-line).
- Trunk-based ou GitHub Flow: escolha um modelo e documente no seu repositório de estudos.

## Testes, design e entrega

- Kent Beck — *Test Driven Development: By Example*.
- Steve Freeman, Nat Pryce — *Growing Object-Oriented Software, Guided by Tests*.
- Michael Feathers — *Working Effectively with Legacy Code* (quando começar a refatorar código “real”).

## Código legível e evolução

- Robert C. Martin — *Clean Code* (com filtro crítico: nem tudo envelhece bem).
- John Ousterhout — *A Philosophy of Software Design*.

## Segurança e operações para devs

- OWASP — [Top 10](https://owasp.org/www-project-top-ten/) e guias de verificação.
- Alice and Bob Learn Application Security (recurso moderno e acessível em inglês).

## Observabilidade (pós-SO/redes)

- Cindy Sridharan — *Distributed Systems Observability* (introdutório).
- Documentação de OpenTelemetry como referência prática.

**Prova de nível sugerida:** um repositório seu com CI rodando testes + README com “como rodar em 3 comandos”.
